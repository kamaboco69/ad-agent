1.目的と方針

手順書に定義された日次・週次・月次の運用ルールを Ad Agent のルールエンジンとして実装し、観測・検知・記録は完全自動、広告への変更操作は「安全なものだけ自動・それ以外は提案止まり」にする。手順書の大原則（計測第一・1変更1検証・記録・データ量・役割分担）をコードの制約として埋め込む。

2.自動化レベルの定義

機能ごとに4段階のレベルを定義し、接続（広告アカウント）単位で設定できるようにする。

レベル意味例
L0 記録観測してログに残すだけ変更ログ、日次スナップショット
L1 検知・提案異常や改善余地を検知してアラート・提案を出す（変更しない）予算消化±10%乖離、CPA目標2倍超、予算再配分案
L2 ワンクリック提案に「適用」ボタンが付き、人が承認して実行除外推奨の一括除外、昇格キーワード登録
L3 全自動条件を満たすものは自動実行し、事後レポートで報告CV0語句の自動除外（実装済み）
既定値の原則: お金が減る方向・学習を壊す方向の操作は L2 まで。L3 に上げられるのは「実施しても損失が出ない」ことが構造的に保証できる操作のみ（例: CV0 語句の除外）。レベル変更は接続ごとのトグルで、すべて変更ログに記録される。

6.安全設計（レッドラインの実装）

  • CVが1件でも付いた語句は自動除外しない（実装済み）。除外は常に上限30件/回。
  • 自動の変更操作は週1回まで・1接続1種類まで（1変更1検証をコードで強制）。
  • 予算・目標値の変更幅は±20%を上限とし、超える指示はUIでも拒否。
  • 学習期間中（直近変更から2週間以内 or CV50件未満）の大変更は警告、blockモードでは実行不可。
  • 自動実行はすべて変更ログ＋AIインサイトに記録され、除外キーワード等は媒体側からいつでも戻せる。
  • 計測異常（CVゼロ検知）中は、自動除外・自動予算変更を一時停止（計測第一の原則）。

7.通知設計

AIインサイト欄を通知センターとして使う（既存）。アラートは重要度（info / warn / critical）を持ち、critical（CVゼロ・予算異常・不承認検知）は将来 LINE / Discord 連携で即時プッシュ可能にする（AI Company の既存 LINE 基盤を流用可能）。

9.自動化の対象外（人間の作業として残るもの）

  • クリエイティブ制作・LP改善・広告文の最終判断（AIは下書き支援まで）
  • Google管理画面でしかできない設定（拡張コンバージョン有効化・入札単価目標値調整ツール・自動適用おすすめのOFF）→ アプリはチェックリストとして未対応を検知・案内
  • 不承認の再審査請求・サポート問い合わせ
  • Customer Match リストの準備（アップロード自体は将来対応可）

10.決めどころ（このレビューで決めてほしいこと）

①目標値の持ち方 — 目標CPA/ROASは接続（アカウント）単位で1つ？ それともキャンペーン単位でも設定したい？（月次の目標対比の粒度が変わる。推奨: まず接続単位）
②自動化レベルの初期値 — 語句の昇格（キーワード登録）を将来 L3（全自動）まで上げたい？（推奨: L2のまま。登録は配信を広げる操作のため）
③critical通知 — CVゼロ・予算異常などの緊急アラートを LINE に飛ばす？（AI Companyのラヴ経由 or Ad Agent独自チャネル。Phase 1に含めるか）
④学習期間ガードの既定 — warn（警告のみ・現状）と block(実行不可)、どちらを既定にする？（推奨: warn。blockは誤検知時に不便）
⑤日次チェックの時刻 — 9:00のみ？ 21:00にも実行して夕方の異常を拾う？（推奨: 9:00のみで開始）
